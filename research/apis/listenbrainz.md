# ListenBrainz API

**Base URL:** https://api.listenbrainz.org/1/
**Docs:** https://listenbrainz.readthedocs.io/en/latest/users/api/index.html
**Auth:** User token (from ListenBrainz account settings) for write; most reads are public
**Cost:** Fully free and open-source (MetaBrainz Foundation)
**Server code:** https://github.com/metabrainz/listenbrainz-server (~300 stars)

## Overview

ListenBrainz is an open-source alternative to Last.fm scrobbling, run by the MetaBrainz
Foundation (the same org behind MusicBrainz). It has several advantages over Last.fm for
a technical project:

- Fully open data model and API — no black-box behavior
- Built-in collaborative filtering recommendation pipeline (Apache Spark)
- Feedback API for rating recommendations (like/dislike loop)
- All entities linked to MusicBrainz MBIDs
- No rate limit surprises; the entire codebase is readable

## Key Endpoints

### Scrobbling

```
POST /1/submit-listens
```
Body: JSON with `listen_type` ("single", "playing_now", "import") and array of listens,
each with `listened_at` timestamp, `track_metadata` (artist, track, album, MBIDs, etc.)

### Listen History

```
GET /1/user/{username}/listens
    ?count=100&max_ts=UNIX_TS&min_ts=UNIX_TS
```
Full paginated history with timestamp cursors. No artificial cap like Spotify's 50-item limit.

```
GET /1/user/{username}/playing-now
GET /1/user/{username}/listen-count          # Total lifetime listen count
```

### Statistics

```
GET /1/stats/user/{username}/artists         # Top artists
    ?range=all_time|this_year|this_month|this_week
GET /1/stats/user/{username}/releases        # Top releases
GET /1/stats/user/{username}/recordings      # Top recordings (tracks)
GET /1/stats/user/{username}/listening-activity   # Listening trends over time
GET /1/stats/user/{username}/daily-activity        # Heatmap by hour/day of week
GET /1/stats/user/{username}/artist-map            # Geographic origin of listened artists
```

### Recommendations (Built-in CF Pipeline)

```
GET /1/cf/recommendation/user/{username}/recording
    ?count=25&offset=0
```
Returns CF-based track recommendations generated by the ListenBrainz Spark pipeline.
Each recommendation includes a `latest_listened_at` field if you've heard it before.

```
POST /1/feedback/recording-feedback
```
Body: `{"recording_msid": "...", "score": 1}` — score is 1 (like) or -1 (dislike).
This feedback is used to improve future recommendations.

```
GET /1/recommendations/cf_recording/feedback/user/{username}
    ?score=1&count=25     # Get all liked/disliked recommendations
```

### Auto-Generated Playlists

ListenBrainz generates several weekly playlists per user:
- "Weekly Jams" — collaborative filtering picks
- "Weekly Exploration" — less-heard tracks in your taste profile
- "Top Discoveries" — new-to-you tracks

```
GET /1/user/{username}/playlists/createdfor    # System-generated playlists
GET /1/playlist/{playlist_mbid}               # Retrieve a playlist's tracks
```

### Social

```
GET /1/user/{username}/followers
GET /1/user/{username}/following
GET /1/social/feed                            # Activity feed
```

### LB Radio

```
GET /1/explore/lb-radio?mode=easy&prompt=artist:(Radiohead)
```
Generates a playlist based on a text prompt (artist/tag/MBID). The "LB Radio" feature
lets you explore by mood, artist, or tag in a programmatic way.

## Recommendation Pipeline Code

The actual recommendation pipeline is open-source:
**https://github.com/metabrainz/troi-recommendation-pipeline** (~100 stars)

Uses Apache Spark + ALS (Alternating Least Squares) matrix factorization on the full
public listen dataset. You can read the code directly to understand the approach,
or run it locally against your own data.

## Assessment for This Project

ListenBrainz is worth using alongside Last.fm for:
1. **Open CF recommendations** — the pipeline is inspectable and the feedback loop is clean
2. **Alternative history store** — can dual-scrobble to both Last.fm and ListenBrainz
3. **Feedback loop study** — their like/dislike → CF update cycle is directly analogous
   to the try-and-rank experiment loop you want to build
4. **Research** — reading the troi pipeline code gives implementation ideas for your own system
